218  // Performs 'docker run IMAGE'. Returns the exit status of the
219  // container. Note that currently the exit status may correspond
220  // to the exit code from a failure of the docker client or daemon
221  // rather than the container. Docker >= 1.10 [1] uses the following
222  // exit statuses inherited from 'chroot':
223  // 125 if the error is with Docker daemon itself.
224  // 126 if the contained command cannot be invoked.
225  // 127 if the contained command cannot be found.
226  // Exit code of contained command otherwise.
227  //
228  // [1]: https://github.com/docker/docker/pull/14012
