56 // Returns a path representation of a ContainerID that can be used for
57 // creating cgroups or writing to the filesystem. A ContainerID can
58 // represent a nested container (i.e, it has a parent ContainerID) and
59 // the path representation includes all of the parents as directories
60 // in the path. Depending on the 'mode', the result can be the
61 // following for a nested container 'xxx.yyy':
62 // 1) mode == PREFIX: '<separator>/xxx/<separator>/yyy'
63 // 2) mode == SUFFIX: 'xxx/<separator>/yyy/<separator>'
64 // 3) mode == JOIN: 'xxx/<separator>/yyy'

71 // The containerizer uses the runtime directory (flag 'runtime_dir')
72 // to checkpoint things for each container, e.g., the PID of the first
73 // process executed within a container (i.e., the "PID 1") gets
74 // checkpointed in a file called 'pid'. The following helper function
75 // constructs the path for a container given the 'runtimeDir' that was
76 // used as well as the container `containerId`. For example, given two
77 // containers, one with ID 'a9dd' and one nested within 'a9dd' with ID
78 // '4e3a' and with the flag 'runtime_dir' set to '/var/run/mesos' you
79 // would have a directory structure that looks like:
80 //
81 // /var/run/mesos/containers/a9dd
82 // /var/run/mesos/containers/a9dd/pid
83 // /var/run/mesos/containers/a9dd/containers/4e3a/pid
84 std::string getRuntimePath(
85  const std::string& runtimeDir,
86  const ContainerID& containerId);
